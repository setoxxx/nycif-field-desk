# Admin Dashboard Data Panels v1

Read-only operator dashboard at `/admin/` that loads static JSON snapshots from `admin/data/`.

## Scope

- **Mode:** read-only visibility only
- **Data sources:** `admin/data/index.json`, `project-status.json`, `source-freshness.json`, `tvpp-candidates.json`, `tvpp-triage.json`, `tvpp-location-readiness.json`
- **No writes:** no publish, approve, reject, geocode, cache, or production feed controls

## Dashboard sections

1. **Header** — title, read-only mode badge, snapshot timestamp, no-publish warning
2. **System Overview** — snapshot availability, freshness, TVPP counts, location readiness, XRI summary, public map review status
3. **Project Status** — current phase, completed safe phases, next recommended step, safety warnings
4. **Source Freshness** — per-source status, date range, recommendations, warnings
5. **TVPP Candidate Panel** — candidate counts, triage buckets, strongest rows, needs-review rows (leads only — not approved assignments)
6. **TVPP Triage** — expandable bucket tables
7. **Location Readiness** — address cleanup, intersection/route, ready/blocked buckets; no geocode approval here
8. **Live Map Review** — links to GitHub Pages Field Desk and `https://nycinfocus.com/map/` plus read-only iframe preview
9. **XRI Status** — G0–G3 complete, G4 held; registry not live-processing production data
10. **Operator Notes** — derived warnings from loaded snapshots

## Safety boundaries

- Does not modify production feed JSON
- Does not modify public map runtime, service worker, WordPress, or iframe/embed settings
- Does not load GPS review artifacts as live data
- Hides sensitive fields (`phone`, `email`, `rawRecord`, tokens/secrets) if present in snapshots
- Missing snapshot files show visible warnings; dashboard remains usable
